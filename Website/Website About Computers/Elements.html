Hardware

Process

The process within a computer system is very simple, your inputs are given to the CPU which tells all of your hardware what to do and what to put on your screen. Some processes would include deleting or creating things on hard drives, opening an executable, or just starting up the computer.

Storage

The storage inside your computer is used to save everything and anything you want, from databases to pictures of your loved ones. There are many types of storage speeds and space, with the most common one today being 500gb.

Control

Control coordinators the input and outputs and the storage on your computer. For example, if you were to delete something, you clicking or pressing delete will send an input to the computer to delete the item, or if you were moving something onto a USB, you would be outputting the data onto a portable storage device.

How a Variety of Devices and CPUs Achieve Their Purpose

There are a variety of input, output, storage, and CPU devices, which means each would achieve its purpose more efficiently than the other, but how would do they achieve their purpose? Here are how some devices achieve its purpose:

  • Keyboard
    • Keyboards achieve their purpose by being able to help put information into documents with efficiency and ease.

  • Monitor
    • Monitors achieve their purpose by projecting valuable information to our screen.

  • SSD
    • SSDs achieve their purpose by storing data and information in a portable storage device.

  • CPU
    • CPUs achieve their purpose as the brain by processing input and output data quickly and effectively with relatively 0 delay.

    Software

    Generations of Programming Languages

    Programming languages have come a long way, getting new versions that refine the old one as tighten the security for a user. there are 4 generations so far, these four are:

  • Machine Code: 1st Generation
    • The 1st Generation of Language (1GL) is a grouping of programming languages that were used to program 1st Generation computers. The instructions were given through the front panel switches of these computers, directly to the CPU. No translator was used to build the 1st Generation Language.

  • Assembly Language: 2nd Generation
    • In the 2nd Generation of Language (2GL), programming languages such as FORTRAN and COBOL were used. The computers used batch processing and multiprogramming operating systems. The difference of this language from 1GL was that the code could be easily read and written very easily by a human.

  • Higher-Level Languages (Imperative/Procedural): 3rd Generation
    • The 3rd Generation of Language (3GL) refers to a high-level programming language such as FORTRAN, COBOL, BASIC, Pascal and C. The difference of this language from 2GL was that it was more machine-dependent and programmer-friendly.

  • Declarative (Non-Procedural) Languages: 4th Generation
    • Most 4th Generation Languages (4GL) are non-procedural languages that encourage users and programmers to specify the results they want, while the computer determines the sequence of instructions that will accomplish those results. The difference from 3GL was that it was an attempt to get closer to human language.

    Functions of Operating Systems

  • Provide Interface to Hardware
    • Operating systems manages computer hardware resources and provides common service for application software. The operating system acts as an interface between the hardware and programs requesting I/O.

  • Provide Interface to User
    • Operating systems provides a UI and an environment for the user to interact with the computer. Without the interface, the user wouldn't be able to see what they are doing which would completely defeat the purpose of having a computer system.

  • Provide Interface to Software Applications
    • System softwares maintains the system resources and gives the path for application software to run.

  • Control the Concurrent Running of Software Applications
    • Concurrency executes multiple instruction sequences at the same time. It happens in the operating system when there are server process threads running.

  • Manage System Resources
    • Operating systems can run multiple tasks at a time and store hundreds of pieces of data with efficiency and ease. Your memory can affect the speed and amount of tasks you run at a time. The only thing your memory can’t affect, is the hardware you use.
